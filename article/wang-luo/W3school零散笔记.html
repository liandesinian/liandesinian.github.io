HTML

HTML用7位ASCII代码在网络上传输数据。实体是用于HTML的，可以用实体名和实体号表示，实体名的格式是：'&' + 字符 + ';', 实体号的格式是：'&#' + 数字(十进制） + ';'。

URL编码是用于URL的，格式为：'%' + 十六进制数, 有些可能会进行二次编码，变成：'%25' + 十六进制数，'%25'是'%'的编码。

<div>是一个块级元素，定义文档的分区，<span>用来组合行内元素。

HTML5应用缓存：在<html>标签中有manifest属性

web worker是运行在后台的Javascript, 不会影响页面的性能

HTML5服务器发送事件允许网页获得来自服务器的更新

HTML注释：<!--... -->

Javascript注释：单行注释：// 代码段注释：/... /

JavaScript

数字和字符串相加，将数字转换成字符串

语句后可加分号，也可以不加。

JavaScript不使用类，不用类来创建对象，而是通过function

window对象表示浏览器窗口，所有Javascript全局变量是window对象的属性，全局函数是window对象的方法。HTML DOM的document是window对象的属性之一。

DOM

HTML DOM是操作、修改、增加、删除HTML元素的标准，是一个编程接口。在HTML DOM中所有事物都是节点。整个文档是文档节点，每个HTML元素是元素节点，元素内的属性是属性节点，元素内的文本是文本节点，注释是注释节点。HTML DOM将HTML文档视为节点树。

PHP

无法在浏览器中查看PHP的源码，只能看到PHP的输出，即HTML。因为PHP是服务端的脚本语言，在输出前已经在服务端执行了。

PHP变量必须以'$'开头

PHP的$_GET和$_POST用于检索表单中的值，比如用户的输入。$_REQUEST变量包含$_GET、$_POST以及$_COOKIE的内容。

include和require语句用于包含其他文件，其中include在发生错误的时候只是生成一个警告，脚本会继续执行，require在发生错误的时候会停止脚本。

可以上传文件，上传后会创建一个临时文件副本，此文件副本在脚本结束时会删除, 若要存储在服务器上，需将它拷贝到另外的位置。

cookie是服务器留在用户计算机的小文件，每当相同的计算机通过浏览器请求页面时，会同时发送cookie。通过PHP可以创建和取回cookie。创建cookie：setcookie(name, value, expire)。在发送cookie时，cookie的值会自动进行URL编码，在取回时进行自动解码。PHP通过$_COOKIE取回cookie值，删除cookie应当使过期事件变更为过去的时间点。（注：setcookie需在之前定义）

PHP session是用于保存和更改用户会话信息的，是存储单一用户会话信息的。Session工作机制：为每个访问者提供一个id（UID)，并基于这个id存储信息。这个id一般是存储在cookie中的, 也可以通过URL进行传输。

开始PHP Session：session_start()位于标签前。通过$_SESSION取得设置和获取某一变量的值。通过unset来释放某一个Session变量。

AJAX

AJAX并不是编程语言，而是一种运用于浏览器的技术，它通过在幕后与web服务器进行交互修改网页部分内容，而不是重新载入网页。它使用浏览器与服务器进行异步数据传输，从服务器处获取少量内容，而不是整个页面。通过HTTP请求向服务器获取信息，使用XML作为接收服务器返回数据的格式，用JavaScript修改页面某部分内容。

XMLHttpRequest用于在后台与服务器进行交互，这意味着可以在不重新加载页面的情况下对页面的部分内容进行更新
